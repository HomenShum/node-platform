# Service blueprint

| Stage | User sees | NodeKit handles backstage |
|---|---|---|
| Orient | Job, time, inputs, safe starting action | Case identity and policy |
| Intake | One necessary choice at a time | Validation and continuous state |
| Working | Milestones and current owner | Agent execution, queues, retries |
| Review | Artifact, diff, evidence, decision | Proposal validation and version check |
| Recovery | What failed, what survived, next choices | Exception isolation and resumability |
| Completion | Canonical result, warnings, export, proof | Receipt, hashes, version history |
