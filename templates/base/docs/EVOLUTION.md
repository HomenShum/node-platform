# Evolution discipline

Record an Evolution Ledger event when work changes a primary workflow, public contract, authority boundary, proof requirement, model route, harness behavior, benchmark conclusion, or downstream guarantee.

The reasoning chain is limitation -> immutable evidence -> scoped assumption -> invariant -> verifier -> verified adoption. Agents may draft this chain but may not certify their own interpretation, proof, or downstream adoption.

Run `nodekit evolution verify` before treating ledger history as Harness Gym or Knowledge Evolution input.
