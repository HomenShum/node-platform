# NodeKit release and certification ladder

Engineering readiness, developer alpha, ease certification, Convex adapter maturity, Convex submission, and backend portability are separate targets. A later product-validation gate must not indefinitely block an earlier engineering decision.

## E0 - Engineering foundation

Status: **ready for independent PR review**.

Requires green CI, reviewed contracts, honest claims, valid evidence manifests, compatibility review, and no unresolved P0/P1 engineering defects. E0 does not claim that unfamiliar people or agents find NodeKit easy.

## E1 - Developer alpha

Status: **incomplete**.

Requires an immutable packaged install, cross-platform smoke, one authentic fresh-agent specialization canary, one revision-bound shareable preview, fresh-user browser completion, an honest limitations page, and no unresolved P0/P1 alpha defects.

## E2 - Ease beta

Status: **incomplete**.

Requires repeated setup measurements, a transparently reported fresh-agent pass rate, formative developer and end-user studies, and repair of every P0/P1 usability finding. Small timing samples report `n`, median, range, and observed maximum, not a misleading p95.

## E3 - Ease certified

Status: **not certified**.

Requires a second confirmation cohort on a frozen repaired candidate, a stable agent success threshold, meaningful timing statistics, deployed first-user proof, and aggregate independent NodeProof verification.

## C1 - Convex adapter proven

Status: **not yet proven**.

Requires one real consumer, two-session reactive state, exactly-once proposal application, conflict and reload safety, revision-bound preview deployment, and a fresh-user receipt.

## C2 - Convex component candidate

Status: **blocked by consumer evidence**.

Requires three materially different consumers, an extracted repeated kernel, `convex-test`, an installable example, typed validators, and an upgrade story.

## C3 - Convex directory submission

Status: **blocked**.

Requires publication authority, a published npm package, final component proof, documentation, and explicit submission approval.

## P1 - Backend portability

Status: **future work**.

Requires a PostgreSQL implementation, a Supabase managed profile, export/import proof, and matching observable Caseflow semantics. Convex-first must never be presented as Convex-only.
