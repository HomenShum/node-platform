Most coding agents stop when the code compiles.

Today we tested a harder standard: carry one consequential workflow from an empty directory through generation, evaluation, a bounded live-provider test, production browser proof, deployment, editable slides, two captioned walkthroughs, and public receipts.

NodeKit v0.2.0 generated two app presets from empty directories. In a deterministic local acceptance run, it installed them, compiled each definition twice, ran their demos, evaluations, and benchmarks, and verified their receipts in 37.2 seconds.

Then we pushed the contract into harder products:

- A synthetic small-business lending deployment lab passed one bounded live-provider call, adversarial browser tests, reload recovery, proposal review, and export/reopen. It is local-ready, not production-proven.

- An Agentic RL research lab exported protected trajectories and a blind heldout evaluation. It is a deterministic behavior-cloning baseline, not gradient RL. Its strict privacy-preserving live route returned 404, so it stopped with no fallback, retry, tokens, or spend.

- Founder Quest Graph is live as a synthetic, read-only product. All 15 hosted checks passed in four isolated desktop/mobile light/dark contexts with zero browser, network, or cross-origin errors. A fresh Windows worktree verified 25/25 artifact hashes. It performs no durable writes or remote Neo4j writes and proves no external approval.

Anthropic's rare-disease research grants gave us an independent external scenario: can NodeKit help a researcher map current work, expose novelty risk, design an executable benchmark, and prepare an application without confusing synthesis with biological proof?

The architecture increasingly resembles Carlos E. Perez's "Graph of Loops": research, build, evaluation, proof, presentation, distribution, feedback, and memory watching one another. The graph stays honest only when it remains anchored to sources, commits, executed tests, exports, public URLs, and real outcomes.

This deck and both walkthroughs are themselves part of the benchmark. Their receipts are verified. Publishing this exact post closes the final external gate; its public URL and screenshot are recorded afterward.
