# NodeKit proof campaign speaker notes

These notes are a draft until the Founder Quest deployment, deck export, video,
and distribution receipts exist. Do not convert planned claims to past tense.

## S1 — NodeKit turns intent into proof-carrying operation

Open on the outcome, not the component inventory. NodeKit is the portable contract
and certification path around an agent application. The factory receipt is the first
anchor.

## S2 — Code is only one edge in the graph

Explain the failure class: a benchmark can call a different planner than production;
a UI can show a plausible answer without durable state; a release can cite a test
that was not run against the deployed revision. NodeKit compiles these paths from the
same definition and records where parity is still absent.

## S3 — The factory is already executable

Use the exact 37.221-second deterministic acceptance result. Two independently
generated candidate repositories were installed, compiled twice for reproducibility,
committed, run, evaluated, benchmarked, and proof-checked. Do not call this a cloud or
production benchmark.

## S4 — The banking-adjacent vertical is governed

Show the synthetic Casca FDE lab. Highlight graph-aware blocker explanation, proposal
review, reload persistence, one bounded live model call, export/reopen, and the explicit
absence of an automated credit decision. State clearly that deployment is the missing
release gate.

## S5 — Learning starts after observability

The present baseline is behavior cloning, not RL. The important product result is the
step trajectory, protected reward, adversarial suite, heldout boundary, and a live route
that failed closed instead of silently weakening its privacy policy. The next scientific
milestone is a larger preregistered corpus and a shadow-only trained-policy comparison.

## S6 — Founder Quest is the user-facing graph

Replace this note with exact production evidence only after the public read-only demo
passes. The product should show applicable quests, a bounded graph neighborhood, a
source-backed answer, authority class, and evidence/proof status.

## S7 — The grant is a real external forcing function

Anthropic's rare-disease grant announcement is an external scenario, not an affiliation.
Explain how NodeKit would construct the latest-research ledger, novelty graph, benchmark,
Claude-credit plan, and application packet while keeping scientific and clinical authority
with qualified researchers and professionals.

## S8 — Loops need external anchors

Name the graph: research, build, evaluation, proof, presentation, distribution, feedback,
and memory. The graph is credible only when it resolves to current sources, exact commits,
fresh-browser evidence, exports, public URLs, and observed response. Invite researchers,
builders, and domain operators to bring one bounded consequential workflow. Close on a
falsifiable promise: the system should expose what passed, what failed, what is local,
and what still needs a human.
